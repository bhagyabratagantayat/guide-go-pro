# Phase 3: Guide Onboarding

<objective>
To implement the document handling logic during Guide registration so that newly registered Guides can submit their Profile Photos, Aadhar, and PAN strings for Admin review, and to verify the Admin verification pipeline.
</objective>

<context>
`authController.js` handles user registration and dynamically clones a `Guide` table entry when `role="guide"`. However, it currently ignores any document payloads. We will extract `profilePhoto`, `aadhar`, and `pan` from `req.body` and persist them to the MongoDB Guide collection. Admin controllers `approveGuide` and `rejectGuide` already exist and just need end-to-end verification.
</context>

<task>
<read_first>
- `backend/controllers/authController.js`
</read_first>
<action>
1. Edit `backend/controllers/authController.js` inside the `register` function. Extract `profilePhoto`, `aadhar`, and `pan` from `req.body`.
2. Inside the `if (role === 'guide') { await Guide.create({ ... }) }` block, pass `profilePhoto`, and construct the `documents: { aadhar, pan }` object.
3. Use a mock test script or `run_command` to post a registration as a Guide with these fields to verify MongoDB correctly ingests them.
4. Ping the `PUT /api/admin/guides/approve/:id` endpoint as an Admin to verify the `isVerified` boolean flips to true.
</action>
<acceptance_criteria>
- `authController.js` actively parses `profilePhoto`, `aadhar`, and `pan` from `req.body` and stores it into `Guide`.
- Admin `approveGuide` successfully changes `isVerified: true`.
</acceptance_criteria>
</task>
