# Phase 2: Admin Dashboard & Locations

<objective>
To build out the missing location management controllers on the Admin side, and ensure the Location search API conforms to the Phase 2 requirements (incorporating regex text-queries). Admin stats will also be verified.
</objective>

<context>
The existing architecture already includes `adminRoutes.js` and `locationController.js`. `adminRoutes.js` has methods to view users, guides, bookings, and fetch mocked statistics. However, requirements `ADM-05` (Admin can add/edit/delete Places) and `LOC-02` (`GET /locations?search=`) are missing logic and need to be implemented.
</context>

<task>
<read_first>
- `backend/controllers/locationController.js`
- `backend/controllers/adminController.js`
- `backend/routes/adminRoutes.js`
</read_first>
<action>
1. Edit `backend/controllers/locationController.js` inside `getLocations`. Extract `req.query.search`. If it exists, use `$regex` on the `name` or `description` fields to filter the returned `Location.find()`.
2. Edit `backend/controllers/adminController.js` to add `addLocation`, `updateLocation`, and `deleteLocation` methods utilizing Mongoose `Location.create`, `findByIdAndUpdate`, and `findByIdAndDelete`.
3. Export these location handlers inside `adminController.js`.
4. Edit `backend/routes/adminRoutes.js` to add `router.post('/locations', adminController.addLocation)`, `router.put('/locations/:id', adminController.updateLocation)`, and `router.delete('/locations/:id', adminController.deleteLocation)`.
</action>
<acceptance_criteria>
- `locationController.js` contains `$regex` filter in `getLocations`.
- `adminController.js` contains `addLocation`, `updateLocation`, and `deleteLocation` implementations.
- `adminRoutes.js` handles POST, PUT, DELETE for `/locations`.
</acceptance_criteria>
</task>

<verification>

- [ ] `GET /locations?search=` returns valid matches.
- [ ] Admin dashboard accurately prints mocked statistical data.
